Greedy Algorithms

Greedy algorithms are algorithms that make decisions based on the current state of the problem. They try to come up with a solution at a given stage and never reverses in case the decision made was not the most optimal.

Some examples of greedy algorithms are

  • Dijkstra's algorithm
  • Prim's algorithm
  • Kruskal's algorithm

Greedy algorithms are often used in the following problems:

  • Scheduling problems
  • Finding the best way to maximize the profit
  • Finding the best way to minimize the cost
  • Finding the best way to minimize the time
  • Finding the best way to minimize the distance

Disadvantage

Greedy algorithms solutions are not always the most optimal. This is because the algorithm is focused on just finding a solution and not on finding the optimal solution.

Steps of a greedy algorithm

  1. Find the node with the highest value
  2. Add it to the solution
  3. Remove it from the graph

Here's another example of a greedy algorithm that finds the minimum number of coins needed to make change for a given amount of money:

Here's an implementation of the greedy algorithm in JavaScript:


    function greedy(coins, amount) {
      const sortedCoins = coins.sort((a, b) => b - a);
      let count = 0;
      for (let coin of sortedCoins) {
        while (amount >= coin) {
          amount -= coin;
          count++;
        }
      }
      return count;
    }

The algorithm visits each coin in the array and subtracts it from the amount until the amount is less than the coin. It then increments the count of the coin.
